任務目標：全面升級食材的單位系統。除了原本的「個(x)」與「百分比(%)」，新增「毫升(ml)」與「公克(g)」單位，並在扣除用量時支援「自訂扣除數值」，為未來的數據統計打下基礎。

請修改現有模組，執行以下重構步驟：

1. 修改 `index.html` (新增食物彈窗 Modal)：
   - 將原本的單位切換改為一個更完整的可愛下拉選單或標籤組，選項包含：
     - `🍏 個數 (x)`
     - `🥛 液體 (ml)`
     - `🥩 重量 (g)`
     - `🥫 醬料 (%)`
   - 當使用者選擇 `ml` 或 `g` 時，數量輸入框的右側要動態顯示對應的單位標籤（如 ml 或 g）。

2. 修改 `js/model.js`：
   - 工廠函數 `createFoodItem` 必須確保寫入兩個核心欄位：
     - `unitType`: 儲存使用的單位字串 ('x', 'ml', 'g', '%')。
     - `originalQuantity`: 紀錄最初買回來的總量（例如 1000）。
     - `quantity`: 目前剩餘的總量（初始值等於 originalQuantity）。

3. 修改 `js/storage.js` 中的 `consumeFood(id, amount)` 函數：
   - 讓 `consumeFood` 函數能額外接收一個 `amount` (扣除量) 參數：
     - 如果 `unitType` 是 'x' 且未傳入 amount，預設扣 1。
     - 如果 `unitType` 是 '%' 且未傳入 amount，預設扣 10。
     - 如果 `unitType` 是 'ml' 或 'g'，則依據傳入的 amount 扣除對應數值（例如傳入 200，則減去 200）。
   - 判定完食邏輯：當 `quantity <= 0` 時，將 `finished` 設為 `true`，`quantity` 設為 0。

4. 修改 `js/ui.js` (卡片雙按鈕與用量輸入優化)：
   - 在食材卡片的右側，我們的雙按鈕組升級為：
     - **按鈕 A (用一點)**：
       - 若單位是 `ml` 或 `g`，點擊「用一點」時，彈出一個精緻可愛的小氣泡或微型輸入框（預設填入常用的 50 或 100），讓使用者可以微調「這次用了多少 ml/g」，確認後扣除。
       - 若單位是 `x` 或 `%`，則維持原本點擊直接扣 1 或扣 10% 的快捷操作。
     - **按鈕 B (全吃完)**：點擊直接將剩餘量歸零，判定完食。
   - 卡片上要漂亮顯示目前的剩餘狀態，例如：`鮮奶 (剩餘 750 / 1000 ml)`。

請確保代碼具備高度擴充性，邏輯嚴謹，且維持日系超可愛的視覺效果。
